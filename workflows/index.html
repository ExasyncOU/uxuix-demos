Deploy Gate + Guardrails
11-Punkte Deploy Gate Checkliste. KEIN Deploy ohne bestandenes Gate. Hotfix (P0): G1+G9+G10+G11 Minimum.

Deploy Gate PFLICHT

  • G1: PROD-Script existiert und ist lesbar
  • G2: TEST-Script existiert (Dual-Script-Pflicht)
  • G3: Testdaten-Snapshot vorhanden (data/snapshots/)
  • G4: Mailverteiler: PROD=Kunde, TEST=nur Bodo
  • G5: Config hat config_version + Changelog
  • G6: Backup erstellt (data/backups/)
  • G7: Metriken-Sammlung im Code
  • G8: Max 200 Zeilen/Script (Module max 300)
  • G9: Keine silent exceptions
  • G10: Keine hardcoded User-Pfade
  • G11: Keine Credentials im Code

Production Safety HARTES VERBOT

  • NIEMALS Pipeline remote als Production starten
  • NIEMALS annehmen dass Env-Var Dry-Run ausloest
  • Fuer DRY-RUN immer dediziertes Script verwenden
  • Wochentag pruefen (nur Mo-Do)
  • Bodo fragen ob Production OK ist
  • Incident 06.03: 702 Amendments produktiv (Freitag!)

Email Geografie REGEL

  • 1 Email = 1 Geografie (Provider sperrt sonst)
  • uxuix-personal: bodo@ = NUR Estland
  • uxuix-automation: automation@ = Finnland/Cloud
  • uxuix-bot: bot@ = Deutschland (Kunden-VMs)

Credential Sync PFLICHT

  • .env auf ALLEN VMs synchron halten
  • credential_vault in Supabase aktualisieren
  • Edge Function Secrets pruefen
  • Supabase Vault (vault.decrypted_secrets)
  • Incident 05.03: Passwort geaendert, .env nicht

Multi-Device REGEL

  • B-Hive = Mr. NexusMind, B-Lab = bodob
  • Bash: $HOME, $USERPROFILE, ~ (nie hardcoded)
  • PowerShell: $env:USERPROFILE, $HOME
  • Config: %USERPROFILE% oder relative Pfade

Autonomie Prinzip VISION

  • 0% Fehlerquote, 0% manuelle Intervention
  • Jeder Agent sammelt Metriken
  • PROD + TEST Dual-Script fuer alle Deployments
  • Historische Daten als Testdaten-Simulation
  • Config-Versionierung + Rollback
